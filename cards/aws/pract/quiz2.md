### Front
What is AWS Service Catalog?

### Back
AWS Service Catalog is a service that allows organizations to create and manage catalogs of IT services approved for use on AWS. It enables:

- Centralized management of approved AWS resources
- Governance and compliance control
- Self-service provisioning for end users
- Standardization of deployments across an organization

<!-- Card End -->

<!-- Card Start -->
### Front
What is AWS Marketplace?

### Back
AWS Marketplace is a curated digital catalog where customers can find, buy, deploy, and manage third-party software, data, and services to build solutions and run their businesses on AWS. Key features include:

- Wide range of software offerings from various categories
- Multiple delivery methods (AMIs, SaaS, etc.)
- Open ecosystem for buyers and sellers

<!-- Card End -->

<!-- Card Start -->
### Front
Compare the primary purposes of AWS Service Catalog and AWS Marketplace.

### Back
AWS Service Catalog:
- Internal organization of approved AWS resources
- Used within a single organization

AWS Marketplace:
- External marketplace for third-party software and services
- Open to all AWS customers

<!-- Card End -->

### Front
Who typically manages AWS Service Catalog?

### Back
IT administrators or cloud operations teams within an organization manage AWS Service Catalog. They are responsible for:

- Creating and managing catalogs of approved products
- Setting up access controls and permissions
- Ensuring compliance with organizational policies

### Front
What types of products can be found in AWS Marketplace?

### Back
AWS Marketplace offers a wide variety of products, including:

- Security solutions
- Business applications
- DevOps tools
- Machine learning and AI software
- Data products
- IoT solutions
- Blockchain applications

Products are available in various formats such as AMIs, SaaS, and container images.

### Front
How does AWS Service Catalog help with governance and compliance?

### Back
AWS Service Catalog aids governance and compliance by:

- Controlling access to approved resources
- Enforcing standardization of deployments
- Allowing administrators to set constraints on product use
- Providing centralized management of IT services
- Enabling tracking and auditing of resource provisioning

<!-- Card End -->

### Front
Can AWS Marketplace products be integrated with AWS Service Catalog?

### Back
Yes, AWS Marketplace products can be integrated with AWS Service Catalog. This integration allows organizations to:

- Add approved Marketplace products to Service Catalog portfolios
- Make third-party solutions available to users through Service Catalog
- Apply organizational governance to Marketplace products
- Streamline procurement and deployment of external solutions

<!-- Card End -->

<!-- Card Start -->
### Front
What is the difference in user base between AWS Service Catalog and AWS Marketplace?

### Back
AWS Service Catalog:
- Used within a single organization
- End users are employees or members of the organization

AWS Marketplace:
- Open to all AWS customers
- Users can be individuals, businesses, or organizations of any size

<!-- Card End -->

### Front
How does AWS Service Catalog enable self-service provisioning?

### Back
AWS Service Catalog enables self-service provisioning by:

- Providing a portal for end users to browse approved products
- Allowing users to launch pre-configured resources
- Enforcing permissions and constraints set by administrators
- Streamlining the deployment process for common IT services
- Reducing the need for manual approvals for standard resources

<!-- Card End -->

<!-- Card Start -->
### Front
What are the benefits of using AWS Marketplace for software procurement?

### Back
Benefits of using AWS Marketplace include:

- Easy discovery of software solutions
- Simplified procurement process
- Integration with existing AWS accounts and billing
- Ability to deploy software directly to AWS environments
- Access to a wide range of pre-configured solutions
- Flexible pricing models (pay-as-you-go, annual subscriptions, etc.)
- Reduced time-to-value for implementing new solutions
